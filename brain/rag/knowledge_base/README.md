# RAG 知识库 · Knowledge Base

本目录用于存放用户自定义的本地 Markdown 资料，供 RAG（检索增强生成）模块在构造 Prompt 时按需挂载。

---

## 目录用途

当 AI 为某位联系人生成回复建议时，`brain/rag/retriever.py` 会根据当前对话的场景标签（商务 / 亲密），
在本目录中检索语义相关的片段，将其注入 Prompt 上下文，使生成结果更贴合用户的真实处境。

---

## 推荐文件组织方式

```
knowledge_base/
├── business/               # 商务场景相关资料
│   ├── project_context.md  # 当前项目背景、技术栈、对接方信息
│   ├── clients.md          # 重要客户 / 合作方关键信息摘要
│   └── action_items.md     # 待跟进事项与里程碑
│
├── intimate/               # 亲密场景相关资料
│   ├── family_events.md    # 家庭重要节点（婚礼进度、纪念日等）
│   ├── shared_memories.md  # 与亲密关系人的共同记忆片段
│   └── pets_hobbies.md     # 宠物护理习惯、兴趣爱好等生活细节
│
└── global/                 # 全场景通用资料
    ├── self_intro.md       # 个人简介与核心人设描述
    └── faq.md              # 自己常被问到的问题及标准回答
```

---

## 使用规范

| 规则 | 说明 |
|------|------|
| **格式** | 纯 Markdown，建议使用二级标题（`##`）分段，便于向量切片 |
| **粒度** | 每个文件聚焦一个主题，避免单文件过长（建议 < 500 行） |
| **更新** | 随时手动添加 / 修改文件，`retriever.py` 在每次会话启动时重新索引 |
| **隐私** | 本目录内容仅在本地使用，不会上传至任何外部服务器 |

---

## 与 RAG 模块的交互

```
用户输入 + 联系人场景标签
        │
        ▼
retriever.py
  ├── 将用户输入编码为语义向量
  ├── 在本目录的已索引文件中检索 Top-K 相关片段
  └── 将片段拼接至 Prompt 的 [CONTEXT] 区域
        │
        ▼
brain/api_hub/router.py → 外部 AI 模型
```

---

> 首次使用时，请根据自身情况在对应子目录下创建 Markdown 文件。
> 文件越丰富，AI 建议的上下文感知能力越强。
